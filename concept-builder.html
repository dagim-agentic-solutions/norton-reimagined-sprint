🎨 Norton Reimagined Design Language — tap to expand

▼
Typography
  • Inter Tight — only typeface
  • Weights: 400 · 500 · 600 · 700 · 800
  • Hero headlines: 800 weight, tight tracking (−0.035em)
  • Body: 400–500, 1.45–1.5 line height
  • Labels: 700, 0.10–0.14em letter-spacing, UPPERCASE
  • -webkit-font-smoothing: antialiased always
Color Palette
  • #242424 — ink (primary text, dark surfaces)
  • #FFE800 — yellow (accent, CTA, highlights)
  • #F8F8F7 — paper (page background)
  • #FFFFFF — card (elevated surfaces)
  • #E6E6E4 — line (borders, dividers)
  • #5a5a5a — ink-soft (secondary text)
  • #8a8a8a — ink-softer (tertiary, labels)
Spacing & Layout
  • Page max-width: 1180px, centered
  • Page padding: 56px (desktop) · 24px (mobile)
  • Section gap: 18–28px between cards
  • Card padding: 26–28px
  • Grid: 12-column, 18px gap
Border Radius
  • 999px — pill buttons, tags, badges, nav items
  • 8px — cards, inputs, standard containers
  • 36px — iOS device frame
  • 6px — small UI elements, code blocks
Component Patterns
  • Nav: dark (#242424) background, yellow bottom border, sticky/fixed
  • Active nav item: yellow pill background, ink text
  • Primary CTA: yellow pill, ink text, 700–800 weight
  • Eyebrow labels: ink bg, yellow text, pill, 11px 600 uppercase
  • Yellow highlight: background: #FFE800; padding: 0 10px; border-radius: 6px;
  • Cards: white bg, 1px #E6E6E4 border, 8px radius
  • Section breaks: 40px h2, 800 weight, −0.025em tracking
iOS Prototype Specifics
  • Canvas: 390×844pt @3x (iPhone 14/15 Pro)
  • Tab bar: pill-style, ink background, yellow active indicator
  • Safe area: 44pt top, 34pt bottom
  • Touch targets: minimum 44×44pt
  • Status first: "You're protected" before any action
  • Light/dark mode toggle required — all text, icons, and the logo must remain fully readable in both modes. Toggle must be accessible from the main screen.

13

First value moment

The first value moment is the earliest point in a mobile app where a new user experiences a clear, concrete benefit that makes them think "this was worth downloading." It's not the onboarding — it's the first time the app does something real for them.

Example: For a weather app, the first value moment is seeing tomorrow's forecast for your exact location within 10 seconds of opening — before you've created an account or adjusted any settings.

For your prototype: what is the single moment — ideally within the first 60 seconds of use — when Laura thinks "this is already helping me"?
